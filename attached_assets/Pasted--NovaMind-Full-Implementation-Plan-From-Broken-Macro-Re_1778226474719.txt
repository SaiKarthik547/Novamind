## PART 1: GAME FIX (nova_mindscape.py + main.py)

### Why the game doesn't start — the exact bug

`nova_mindscape.py` `run()` initialises Ursina but returns immediately.
`main.py` starts a daemon thread calling `game.run()` then drives it via Qt timer calling `game.step()`.

**The problem**: Ursina requires its OWN mainloop (`app.run()`) to process OpenGL frames, input events, and entity updates. `application.step()` is only valid INSIDE an already-running Ursina loop. Calling it from a Qt timer when `app.run()` was never called does nothing — there is no frame pipeline to step.

Qt and Ursina both want to own the main thread. They cannot share it. The current code tries to hack around this and fails silently.

### Fix: Run Ursina in a fully isolated process

```python
# game/nova_mindscape_launcher.py  — NEW FILE
"""
Runs NovaMind's Ursina game in a completely separate process.
Communicates with the main process via a multiprocessing Queue.
This is the only correct way to run Ursina alongside Qt.
"""
import multiprocessing
import time
import logging

logger = logging.getLogger("GameLauncher")

def _game_process_main(task_queue: multiprocessing.Queue, config_dict: dict):
    """Entry point for the game subprocess."""
    try:
        from game.nova_mindscape import NovaMindscape, GameConfig
        config = GameConfig(**config_dict)
        game = NovaMindscape(config=config)
        game._task_queue = task_queue
        game.run_blocking()   # NEW method — calls app.run() which blocks
    except Exception as e:
        logger.error(f"Game process crashed: {e}")


class GameProcessManager:
    """Manages the Ursina game as a subprocess."""

    def __init__(self, config_dict: dict):
        self._config = config_dict
        self._queue = multiprocessing.Queue(maxsize=50)
        self._process = None

    def start(self):
        self._process = multiprocessing.Process(
            target=_game_process_main,
            args=(self._queue, self._config),
            daemon=True,
            name="nova-game"
        )
        self._process.start()
        logger.info(f"Game process started (PID {self._process.pid})")

    def update_tasks(self, tasks: list):
        """Thread-safe — called from Brain callbacks."""
        try:
            # Non-blocking put — drop if queue full
            self._queue.put_nowait({"type": "tasks", "data": tasks})
        except Exception:
            pass

    def stop(self):
        if self._process and self._process.is_alive():
            self._process.terminate()
            self._process.join(timeout=3)
```




### New file: `core/os_executor.py`

This replaces the broken pyautogui-only approach with a layered execution system.

```python
"""
core/os_executor.py

The real OS-level execution layer. Three execution strategies, tried in order:
1. Windows UI Automation (UIA) — semantic element finding, most reliable
2. Vision-guided (OCR + template match) — when UIA fails
3. Coordinate-based (pyautogui) — last resort, with full safety guards

All strategies share:
- Active-window assertion before every action
- DPI-aware coordinate translation
- Modifier key release before scroll/click
- Mouse-up in finally blocks
- Before/after screenshot verification
"""

import ctypes
import time
import logging
import pyautogui
import pygetwindow as gw
from typing import Optional, Tuple, Dict, Any
from PIL import Image, ImageGrab

logger = logging.getLogger("OSExecutor")

# ── DPI handling ──────────────────────────────────────────────────────────────

def get_dpi_scale() -> float:
    """Get the real DPI scale factor from Windows."""
    try:
        # Make process DPI-aware first
        ctypes.windll.user32.SetProcessDPIAware()
        scale = ctypes.windll.shcore.GetScaleFactorForDevice(0) / 100.0
        return scale
    except Exception:
        return 1.0

DPI_SCALE = get_dpi_scale()

def logical_to_physical(x: int, y: int) -> Tuple[int, int]:
    """Convert pygetwindow logical coords → pyautogui physical coords."""
    return int(x / DPI_SCALE), int(y / DPI_SCALE)

def physical_to_logical(x: int, y: int) -> Tuple[int, int]:
    return int(x * DPI_SCALE), int(y * DPI_SCALE)

# ── Focus guard ───────────────────────────────────────────────────────────────

class FocusLostError(Exception):
    pass

def assert_window_focused(title_contains: str, max_wait: float = 2.0) -> bool:
    """
    Verify the target window has focus. Re-focus if lost. Raise if unrecoverable.
    Call this before EVERY pyautogui action.
    """
    deadline = time.time() + max_wait
    while time.time() < deadline:
        try:
            active = gw.getActiveWindow()
            if active and title_contains.lower() in active.title.lower():
                return True
        except Exception:
            pass

        # Try to bring target window to front
        try:
            wins = gw.getWindowsWithTitle(title_contains)
            if wins:
                wins[0].activate()
                time.sleep(0.15)
        except Exception:
            time.sleep(0.1)

    raise FocusLostError(
        f"Could not establish focus on window containing '{title_contains}' "
        f"after {max_wait}s — aborting to prevent wrong-app execution"
    )

# ── Modifier key safety ───────────────────────────────────────────────────────

def release_all_modifiers():
    """Release all modifier keys. Call before scroll and before drag start."""
    for key in ('ctrl', 'shift', 'alt', 'win'):
        try:
            pyautogui.keyUp(key)
        except Exception:
            pass

# ── Canvas / region detection ─────────────────────────────────────────────────

def detect_paint_canvas() -> Optional[Tuple[int, int, int, int]]:
    """
    Dynamically detect the MS Paint canvas bounding box.
    Returns (left, top, right, bottom) in physical screen coordinates.
    Strategy: screenshot the Paint window top 250px, find where the
    white canvas region starts by scanning for a run of white pixels.
    """
    try:
        wins = gw.getWindowsWithTitle("Paint")
        if not wins:
            return None
        w = wins[0]

        # Convert to physical coords
        wx, wy, ww, wh = w.left, w.top, w.width, w.height
        wx_p, wy_p = logical_to_physical(wx, wy)
        ww_p = int(ww / DPI_SCALE)
        wh_p = int(wh / DPI_SCALE)

        # Screenshot top 260px of Paint window to find ribbon bottom
        ribbon_img = ImageGrab.grab(bbox=(wx_p, wy_p, wx_p + ww_p, wy_p + 260))
        ribbon_arr = list(ribbon_img.getdata())
        width = ribbon_img.width

        # Scan downward looking for a row that is >70% white pixels
        canvas_top_local = 125  # fallback
        for row in range(80, 260):
            row_pixels = ribbon_arr[row * width:(row + 1) * width]
            white_count = sum(
                1 for r, g, b in row_pixels
                if r > 240 and g > 240 and b > 240
            )
            if white_count / width > 0.70:
                canvas_top_local = row
                break

        return (
            wx_p + 4,
            wy_p + canvas_top_local,
            wx_p + ww_p - 4,
            wy_p + wh_p - 30,
        )
    except Exception as e:
        logger.warning(f"Canvas detection failed: {e}")
        return None

def point_inside_region(x: int, y: int,
                         region: Tuple[int, int, int, int]) -> bool:
    l, t, r, b = region
    return l <= x <= r and t <= y <= b

# ── Safe action primitives ────────────────────────────────────────────────────

def safe_click(x: int, y: int, window_title: str,
               button: str = 'left', clicks: int = 1):
    assert_window_focused(window_title)
    release_all_modifiers()
    pyautogui.click(x, y, button=button, clicks=clicks, interval=0.1)

def safe_drag(x1: int, y1: int, x2: int, y2: int,
              window_title: str, duration: float = 0.4):
    assert_window_focused(window_title)
    release_all_modifiers()
    try:
        pyautogui.mouseDown(x1, y1)
        time.sleep(0.05)
        pyautogui.moveTo(x2, y2, duration=duration)
    finally:
        pyautogui.mouseUp()  # ALWAYS release even on crash/timeout

def safe_scroll(x: int, y: int, clicks: int, window_title: str,
                region: Optional[Tuple[int, int, int, int]] = None):
    assert_window_focused(window_title)
    release_all_modifiers()  # CRITICAL: prevents Ctrl+Scroll zoom
    if region and not point_inside_region(x, y, region):
        raise ValueError(f"Scroll target ({x},{y}) outside safe region {region}")
    pyautogui.scroll(clicks, x=x, y=y)

def safe_type(text: str, window_title: str, interval: float = 0.05):
    assert_window_focused(window_title)
    pyautogui.typewrite(text, interval=interval)

def safe_hotkey(window_title: str, *keys):
    assert_window_focused(window_title)
    release_all_modifiers()
    pyautogui.hotkey(*keys)

# ── Screenshot verification ───────────────────────────────────────────────────

def capture_region(region: Tuple[int, int, int, int]) -> Image.Image:
    return ImageGrab.grab(bbox=region)

def images_differ(img1: Image.Image, img2: Image.Image,
                  threshold: float = 0.005) -> bool:
    """Returns True if images differ by more than threshold fraction of pixels."""
    try:
        import numpy as np
        a1 = np.array(img1.convert('RGB'))
        a2 = np.array(img2.convert('RGB'))
        if a1.shape != a2.shape:
            return True
        diff = np.abs(a1.astype(int) - a2.astype(int))
        changed = np.sum(diff > 15)
        total = a1.size
        return (changed / total) > threshold
    except ImportError:
        # Fallback without numpy
        return img1.tobytes() != img2.tobytes()

class ActionVerifier:
    """
    Wraps any action with before/after screenshot comparison.
    Raises if the action had no visible effect when one was expected.
    """
    def __init__(self, region: Tuple[int, int, int, int]):
        self.region = region
        self._before: Optional[Image.Image] = None

    def __enter__(self):
        self._before = capture_region(self.region)
        return self

    def __exit__(self, exc_type, exc_val, exc_tb):
        return False  # don't suppress exceptions

    def verify_changed(self, description: str = "") -> bool:
        after = capture_region(self.region)
        changed = images_differ(self._before, after)
        if not changed:
            logger.warning(f"Action had no visible effect: {description}")
        return changed

    def verify_unchanged(self, description: str = "") -> bool:
        after = capture_region(self.region)
        return not images_differ(self._before, after)
```

### New file: `core/uia_executor.py` — Windows UI Automation

This is what makes it truly OS-level. UIA lets you find elements by their semantic meaning (button named "OK", text field in window "Paint") without any coordinates.

```python
"""
core/uia_executor.py

Windows UI Automation (UIA) wrapper.
Finds elements by name, control type, automation ID — not by pixel position.
This is how professional automation tools (Playwright, WinAppDriver) work internally.

Requires: pip install comtypes  (pure Python, no extra install on Windows)
"""

import logging
import time
from typing import Optional, List

logger = logging.getLogger("UIAExecutor")

try:
    import comtypes.client as cc
    import comtypes.gen.UIAutomationClient as UIA_client

    _uia = cc.CreateObject(
        "{ff48dba4-60ef-4201-aa87-54103eef594e}",
        interface=UIA_client.IUIAutomation,
    )
    UIA_OK = True
except Exception as e:
    UIA_OK = False
    logger.warning(f"UIA not available: {e} — falling back to vision-guided execution")


class UIElement:
    """Wraps a UIA AutomationElement with convenience methods."""

    def __init__(self, element):
        self._el = element

    @property
    def name(self) -> str:
        try:
            return self._el.CurrentName or ""
        except Exception:
            return ""

    @property
    def rect(self):
        """Returns (left, top, right, bottom) bounding rect."""
        try:
            r = self._el.CurrentBoundingRectangle
            return (r.left, r.top, r.right, r.bottom)
        except Exception:
            return None

    @property
    def center(self):
        r = self.rect
        if r:
            return ((r[0] + r[2]) // 2, (r[1] + r[3]) // 2)
        return None

    def click(self):
        """Click via UIA invoke pattern — no coordinates needed."""
        try:
            pattern = self._el.GetCurrentPattern(UIA_client.UIA_InvokePatternId)
            if pattern:
                pattern.QueryInterface(UIA_client.IUIAutomationInvokePattern).Invoke()
                return True
        except Exception:
            pass
        # Fallback to coordinate click
        c = self.center
        if c:
            import pyautogui
            pyautogui.click(*c)
            return True
        return False

    def set_value(self, text: str):
        """Set text value via UIA ValuePattern."""
        try:
            pattern = self._el.GetCurrentPattern(UIA_client.UIA_ValuePatternId)
            if pattern:
                pattern.QueryInterface(
                    UIA_client.IUIAutomationValuePattern
                ).SetValue(text)
                return True
        except Exception:
            pass
        return False

    def get_value(self) -> str:
        try:
            pattern = self._el.GetCurrentPattern(UIA_client.UIA_ValuePatternId)
            if pattern:
                return pattern.QueryInterface(
                    UIA_client.IUIAutomationValuePattern
                ).CurrentValue or ""
        except Exception:
            pass
        return ""

    def is_enabled(self) -> bool:
        try:
            return bool(self._el.CurrentIsEnabled)
        except Exception:
            return False

    def children(self) -> List['UIElement']:
        try:
            walker = _uia.ControlViewWalker
            children = []
            child = walker.GetFirstChildElement(self._el)
            while child:
                children.append(UIElement(child))
                child = walker.GetNextSiblingElement(child)
            return children
        except Exception:
            return []


class UIAExecutor:
    """
    Finds and interacts with UI elements by semantic identity.
    No hardcoded pixel coordinates.
    """

    def find_window(self, title_contains: str,
                    timeout: float = 5.0) -> Optional[UIElement]:
        """Find a top-level window by title substring."""
        if not UIA_OK:
            return None
        deadline = time.time() + timeout
        while time.time() < deadline:
            try:
                root = _uia.GetRootElement()
                condition = _uia.CreatePropertyCondition(
                    UIA_client.UIA_NamePropertyId, title_contains
                )
                # Try exact first, then walk children for substring match
                el = root.FindFirst(
                    UIA_client.TreeScope_Children, condition
                )
                if el:
                    return UIElement(el)
                # Substring walk
                all_wins = root.FindAll(
                    UIA_client.TreeScope_Children,
                    _uia.CreateTrueCondition()
                )
                for i in range(all_wins.Length):
                    w = all_wins.GetElement(i)
                    if title_contains.lower() in (w.CurrentName or "").lower():
                        return UIElement(w)
            except Exception as e:
                logger.debug(f"find_window error: {e}")
            time.sleep(0.2)
        return None

    def find_element(self, window: UIElement, name: str = None,
                     control_type: int = None,
                     automation_id: str = None) -> Optional[UIElement]:
        """Find element within a window by any combination of properties."""
        if not UIA_OK or not window:
            return None
        try:
            conditions = []
            if name:
                conditions.append(
                    _uia.CreatePropertyCondition(
                        UIA_client.UIA_NamePropertyId, name
                    )
                )
            if control_type:
                conditions.append(
                    _uia.CreatePropertyCondition(
                        UIA_client.UIA_ControlTypePropertyId, control_type
                    )
                )
            if automation_id:
                conditions.append(
                    _uia.CreatePropertyCondition(
                        UIA_client.UIA_AutomationIdPropertyId, automation_id
                    )
                )

            if not conditions:
                return None

            cond = conditions[0]
            for c in conditions[1:]:
                cond = _uia.CreateAndCondition(cond, c)

            el = window._el.FindFirst(UIA_client.TreeScope_Descendants, cond)
            return UIElement(el) if el else None
        except Exception as e:
            logger.debug(f"find_element error: {e}")
            return None

    def find_all_elements(self, window: UIElement,
                          control_type: int = None) -> List[UIElement]:
        """Find all elements of a given type within a window."""
        if not UIA_OK or not window:
            return []
        try:
            cond = (
                _uia.CreatePropertyCondition(
                    UIA_client.UIA_ControlTypePropertyId, control_type
                ) if control_type else _uia.CreateTrueCondition()
            )
            all_els = window._el.FindAll(UIA_client.TreeScope_Descendants, cond)
            return [UIElement(all_els.GetElement(i))
                    for i in range(all_els.Length)]
        except Exception:
            return []

    def get_focused_element(self) -> Optional[UIElement]:
        try:
            el = _uia.GetFocusedElement()
            return UIElement(el) if el else None
        except Exception:
            return None
```

## PART 3: REWRITE application_agent.py — The Right Way

The entire `open_paint_and_draw()` function needs to be replaced with a proper execution pipeline that uses the above infrastructure.

### New Paint execution pipeline

```python
# In agents/application_agent.py — replace open_paint_and_draw() entirely

from core.os_executor import (
    OSExecutor, assert_window_focused, detect_paint_canvas,
    safe_drag, safe_click, safe_hotkey, release_all_modifiers,
    ActionVerifier, FocusLostError, DPI_SCALE
)
from core.uia_executor import UIAExecutor

class ApplicationAgent:

    def __init__(self):
        self._os = OSExecutor()
        self._uia = UIAExecutor()
        # Set pyautogui safety
        pyautogui.FAILSAFE = True
        pyautogui.PAUSE = 0.05

    def execute_paint_task(self, description: str) -> Dict:
        """
        Full pipeline for any Paint drawing task.
        1. Open Paint if not running
        2. Maximise + wait for stable
        3. Detect canvas dynamically
        4. Generate drawing plan from LLM
        5. Execute with verification at every step
        6. Return result with screenshot proof
        """
        try:
            # Step 1: Ensure Paint is open and focused
            paint_win = self._ensure_paint_open()
            if not paint_win:
                return {"success": False, "error": "Could not open MS Paint"}

            # Step 2: Detect canvas (dynamic, DPI-aware)
            canvas = detect_paint_canvas()
            if not canvas:
                return {"success": False, "error": "Could not detect canvas bounds"}

            cx1, cy1, cx2, cy2 = canvas
            canvas_w = cx2 - cx1
            canvas_h = cy2 - cy1

            # Step 3: Get drawing plan from LLM
            plan = self._generate_drawing_plan(description, canvas_w, canvas_h)

            # Step 4: Execute plan with verification
            results = []
            for step in plan:
                result = self._execute_drawing_step(step, canvas, "Paint")
                results.append(result)
                if not result.get("success"):
                    break

            # Step 5: Final verification
            final_screenshot = pyautogui.screenshot(
                region=(cx1, cy1, canvas_w, canvas_h)
            )
            screenshot_path = self._save_screenshot(final_screenshot, "paint_result")

            return {
                "success": all(r.get("success") for r in results),
                "steps_executed": len(results),
                "screenshot": screenshot_path,
            }

        except FocusLostError as e:
            return {"success": False, "error": f"Focus lost: {e}"}
        except Exception as e:
            logger.exception(f"Paint task failed: {e}")
            return {"success": False, "error": str(e)}

    def _ensure_paint_open(self):
        """Open Paint if not running. Return when focused and maximised."""
        import subprocess

        # Check if Paint already open
        wins = pygetwindow.getWindowsWithTitle("Paint")
        if not wins:
            subprocess.Popen(["mspaint"])
            time.sleep(1.5)

        # Wait for Paint window
        for _ in range(20):
            wins = pygetwindow.getWindowsWithTitle("Paint")
            if wins:
                break
            time.sleep(0.3)
        else:
            return None

        # Focus and maximise
        try:
            wins[0].activate()
            time.sleep(0.3)
            pyautogui.hotkey('win', 'up')
            time.sleep(0.5)
            assert_window_focused("Paint", max_wait=3.0)
            return wins[0]
        except Exception as e:
            logger.error(f"Could not focus Paint: {e}")
            return None

    def _execute_drawing_step(self, step: Dict, canvas: Tuple,
                               window_title: str) -> Dict:
        """
        Execute a single drawing step with full safety guards.
        Each step has: type (line/rect/circle/text), coordinates, color
        """
        cx1, cy1, cx2, cy2 = canvas
        step_type = step.get("type")

        with ActionVerifier(region=canvas) as verifier:
            try:
                assert_window_focused(window_title)

                if step_type == "line":
                    # Convert normalised 0-1 coords to absolute pixels
                    x1 = cx1 + int(step["x1"] * (cx2 - cx1))
                    y1 = cy1 + int(step["y1"] * (cy2 - cy1))
                    x2 = cx1 + int(step["x2"] * (cx2 - cx1))
                    y2 = cy1 + int(step["y2"] * (cy2 - cy1))
                    safe_drag(x1, y1, x2, y2, window_title)

                elif step_type == "set_color":
                    self._set_paint_color_uia(step.get("color", (0, 0, 0)))

                # ... other step types

                changed = verifier.verify_changed(f"step: {step_type}")
                return {"success": True, "verified": changed}

            except FocusLostError as e:
                return {"success": False, "error": str(e)}
            except Exception as e:
                pyautogui.mouseUp()  # Safety: release mouse
                return {"success": False, "error": str(e)}

    def _set_paint_color_uia(self, rgb: Tuple[int, int, int]):
        """
        Set Paint color using UIA — no hardcoded pixel ratios.
        Finds the 'Edit colors' button by automation ID, not by coordinate.
        """
        uia = UIAExecutor()
        paint_win = uia.find_window("Paint", timeout=3.0)
```

```python
    result = self._call_agent(node)
    if result.get("success"):
        return result

    # Build recovery context
    context = RecoveryContext(
        error=result.get("error", ""),
        original_task=node.to_dict(),
        attempt=attempt,
    )
    plan = await self.recovery_agent.recover(context)

    if plan and plan.modified_task:
        # CRITICAL: re-dispatch the MODIFIED task, not the original
        modified_node = TaskNode.from_dict(plan.modified_task)
        node = modified_node   # Next iteration uses different parameters
        logger.info(f"Recovery plan applied: {plan.strategy}")
    else:
        logger.warning("Recovery returned no modified task — retry is identical")
        break

return {"success": False, "error": "All recovery attempts exhausted"}
```

```python
def _fallback_parse(self, user_request: str) -> TaskPlan:
    """
    When LLM parsing fails, generate a valid single-step plan.
    The step uses an action that the target agent actually handles.
    """
    agent = self._classify_agent(user_request)
    template = _FALLBACK_ACTION_MAP.get(agent, {
        "action": "run_command",
        "parameters": {"command_line": user_request}
    })

        "parameters": {k: user_request if v is None else v
                       for k, v in template["parameters"].items()},
        "agent": agent,
        "description": user_request,
        "risk_level": 1,
        "timeout": 60,
    }
    return TaskPlan(steps=[step], original_request=user_request)
```

## PART 6: WHAT "OS-LEVEL" ACTUALLY REQUIRES

To be a real OS-level agent, not a macro recorder, you need these capabilities:

### 6.1 Semantic element targeting (not coordinates)

You need a unified element finder that tries, in order:
1. **UIA** (Windows UI Automation) — semantic find by role/name/ID
2. **OCR** — find text on screen, click its bounding box
3. **Template matching** — find a known image pattern (button icon, etc.)
4. **Coordinate** — absolute last resort with full safety guards

```python
# core/element_finder.py

class ElementFinder:
    """
    Find any UI element by semantic description.
    Used by ALL agents, not just ApplicationAgent.
    """

    def find(self, description: str, window_title: str = None,
             strategy: str = "auto") -> Optional[FoundElement]:
        strategies = {
            "uia":      self._find_by_uia,
            "ocr":      self._find_by_ocr,
            "template": self._find_by_template,
        }
        if strategy == "auto":
            for name, fn in strategies.items():
                result = fn(description, window_title)
                if result:
                    logger.info(f"Found '{description}' via {name}")
                    return result
        else:
            return strategies.get(strategy, lambda *_: None)(
                description, window_title
            )
        return None
```

### 6.2 Application-aware action sets

Every app needs its own action module — not one giant ApplicationAgent.

### 6.4 Task planning that generates real executable steps

The LLM currently generates vague steps like `{"action": "draw", "description": "draw a car"}`. It needs to generate concrete steps that the executor actually understands:


## Summary: What "OS-level" Means in Practice

| Current | Required |
|---|---|
| Hardcoded pixel coords | UIA semantic element finding |
| No focus verification | Focus asserted before every action |
| DPI scale = 1.0 always | Real DPI from Windows API |
| Canvas = window.top + 125px | Dynamic canvas detection via pixel scanning |
| Ctrl+Scroll causes zoom | Modifier keys released before every scroll |
| Recovery plan discarded | Modified task re-dispatched into pipeline |
| Verifier never called | Before/after diff on every action |
| One giant ApplicationAgent | Per-app agents with shared execution layer |
| asyncio.run() in Qt thread crashes | Dedicated async loop thread |
| Game subprocess blocked/never starts | Game in separate process via multiprocessing |
| No tests | Unit + integration + E2E coverage |
| Magic numbers everywhere | Central config.py |
